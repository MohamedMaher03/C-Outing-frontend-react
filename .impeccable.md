## Design Context

### Users

- Primary users are Cairo locals who open the app whenever they want to go out and have fun.
- Usage context is often quick, decision-focused moments, so discovery and decision support must feel fast and low-effort.
- Core job to be done is onboarding completion first, so the system can personalize recommendations right away.

### Brand Personality

- 3-word personality: Friendly, Smart, Local.
- Voice and tone should feel welcoming and supportive, like a helpful local friend with strong recommendations.
- Emotional goal should prioritize delight, while still feeling trustworthy and easy to use.

### Aesthetic Direction

- Keep a logo-led visual system using the existing navy/gold/cream direction already present in the product.
- Preserve the current premium-local feel: Cairo imagery, warm accents, and refined surfaces, but avoid generic travel UI clones.
- Reference point: Tripadvisor for discovery clarity and utility, while maintaining a unique C-Outing look and interaction style.
- Product goal is to support both light and dark modes with consistent brand identity and contrast quality.

### Design Principles

1. Logo-Led Consistency: Use logo-derived colors as the primary visual anchor across all features and states.
2. Onboarding-First Clarity: Prioritize clear, low-friction onboarding so users reach personalized value quickly.
3. Local-Expert Warmth: Write and design interactions like a friendly, smart Cairo guide, not a generic booking engine.
4. Delight With Purpose: Use motion and visual polish to create delight, but only when it improves comprehension or feedback.
5. Accessibility As Baseline: Target WCAG AA by default, with strong keyboard support, readable contrast, and reduced-motion-safe behavior.
